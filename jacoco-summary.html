Que significa cada dato

DatoQue mideComo interpretarlo
Cobertura global (93.8%) Porcentaje de lineas de codigo que fueron ejecutadas al menos una vez durante la suite de tests. Un valor alto indica que los tests recorren la mayoria del codigo. Por debajo del 80% suele considerarse insuficiente para proyectos en produccion.
Metodos cubiertos (88.5%) Proporcion de metodos/funciones que fueron invocados por algun test. Complementa la cobertura de lineas: un metodo puede estar cubierto en lineas pero nunca llamado directamente. Valores altos aqui indican una suite robusta.
Lineas cubiertas por paquete Cobertura de lineas desagregada por capa de la aplicacion (controller, service, model, etc.). Permite identificar que capas estan bien testeadas y cuales necesitan atencion. Verde ≥ 80%, amarillo 50–79%.
Lineas cubiertas por clase Cobertura individual de cada clase dentro del proyecto. Util para detectar clases especificas con baja cobertura. Una clase en amarillo no necesariamente es un problema si sus caminos no cubiertos son poco probables en produccion.
Suite de tests ✓ Estado de ejecucion de todos los tests al momento de generar el reporte. Si algun test falla el build no pasa y el reporte no se genera. El check verde confirma que todos los tests pasaron correctamente.
Paquetes excluidos config, dto y repository fueron removidos del calculo. config no tiene logica testeable. dto tiene builders generados por Lombok que inflarian el denominador. repository son interfaces del framework, no codigo propio.
